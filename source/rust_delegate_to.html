2 1. 为什么需要构建， 而不是采用Deref trait?

2.1 因为Deref 中的target 只能有一个， 不能在一个 包含多个field的 struct中，同时 实现 Deref<One> , Deref<Two>, 即如下：

struct FatherInner {

} impl FatherInner { fn test() {} }

struct WorkerOuter {

} impl WorkerOuter { fn fuck() {} }

#[derive(TestMacro)] struct Person { inner: FatherInner, outer: WorkerOuter, } impl Deref for Person { type Target = FatherInner; fn deref(&self) -> Self::Target { &self.inner } }

//error, conflict impl Deref impl Deref for FatherInner { type Target = FatherInner; }

impl Person { }

3 3. 如何构建？ 借助trait 实现不太可能，因为 Deref 是建立在compiler 自动进行deref 函数调用的基础上，所以： 需要借助 macro构建

3.1 初步尝试：

let inputs = ast.sig.inputs; let output = ast.sig.output; let gen = quote! { fn #fn_name() -> String { format!("{} {}", stringify!(#inputs), stringify!(#output)) } fn #origin_fn_name(#inputs) #output { format!("{} {}", stringify!(#inputs), stringify!(#output)); #block } }; gen.into() }
